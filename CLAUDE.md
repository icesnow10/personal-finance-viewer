## Plugin dependency

This viewer works in tandem with the **[open-personal-finance](https://github.com/icesnow10/personal-finance)** Claude Code plugin, which handles fetching bank transactions (via Pluggy API), classifying expenses, recognizing income, and generating the budget JSON files this app consumes.

Before running any financial analysis (`/compile`, `/heartbeat`, `/fetch`, etc.), always update the plugin to get the latest classification rules and skill improvements:

```
/plugin marketplace add icesnow10/personal-finance
/reload-plugins
```

## Architecture

**Next.js Pages Router** app. All pages are in `src/pages/`, API routes in `src/pages/api/`.

### Data flow

1. Budget JSON files live in `resources/{YYYY-MM}/expenses/result/budget_{MM}_{YYYY}.json` (gitignored, generated by the [personal-finance](https://github.com/icesnow10/personal-finance) plugin).
2. `GET /api/budgets` scans the resources directory, reads all valid budget JSONs, and returns them sorted by month. The resources path defaults to `{cwd}/resources` but can be overridden via `.config.json` (also gitignored): `{ "resourcesPath": "/absolute/path" }`.
3. `BudgetContext` (`src/context/BudgetContext.tsx`) fetches from `/api/budgets` on mount, exposes `allMonths: BudgetData[]`, `data: BudgetData` (most recent month), and `flatTransactions: FlatTransaction[]`.
4. Pages consume data via `useBudget()` hook.

### Key types (`src/lib/types.ts`)

`BudgetData` is the central type ‚Äî every budget JSON must conform to it. Key fields:
- `income.items[]` ‚Äî income transactions (salary, dividends, reimbursements)
- `expenses.by_category` ‚Äî `Record<CategoryName, CategoryData>` ‚Üí subcategories ‚Üí transactions
- `expenses.unclassified[]` ‚Äî uncategorized transactions
- `summary` ‚Äî pre-computed totals (income, expenses, net, investment, investment_pct)
- `budget_buckets` ‚Äî exactly 3 keys: `custos_fixos`, `conforto`, `liberdade_financeira`
- `skipped[]` ‚Äî internal transfers excluded from totals
- `intentional_rdb_investments` ‚Äî large lump-sum investments tracked separately, NOT in expense totals
- `partial: true` + `data_through` ‚Äî marks months with incomplete data
- `status` (`"posted" | "pending"`) ‚Äî per-row flag from the plugin. Credit charges are `pending` while the bill is open, `posted` once it closes. Carried through `normalizeBudget` (baseTx allow-list), `flattenTransactions`, and `FlatTransaction`. Missing status defaults to `posted`. Surfaced as a **Status** column + inline "Pendente" tag in the transactions table, and as a top-right warning badge on the overview **Resultado Parcial** card when any pending rows remain.

### Computation layer (`src/lib/computations.ts`)

Pure functions over `BudgetData` used by pages:
- `getSpendingPace` ‚Äî daily avg, projected total, comparison vs previous month
- `getBucketProgress` ‚Äî recalculates bucket amounts from `expenses.by_category` (source of truth is the category data, not the pre-computed bucket fields in the JSON)
- `getDailySpendingCurve` ‚Äî cumulative spending by day for chart rendering
- `getRecurringTransactions` ‚Äî finds transactions appearing in ‚â•2 months

### Responsive layout

`useIsMobile` (`src/hooks/useIsMobile.ts`) is a `matchMedia(max-width: 767px)` hook used across pages. `AppShell` switches to a mobile layout below 768px: the left `Sider` is hidden and a fixed bottom navigation (`/overview`, `/transactions`, `/investments`, `/settings`) replaces it. `Content` adds bottom padding equal to the nav height (64px) plus `env(safe-area-inset-bottom)`. Pages collapse two-column grids to a single column at the same breakpoint and `VisorCard` reduces internal padding.

### Scheduled prompts (top-bar modal)

Terminal icon in the header opens `PromptShortcutModal` (`src/components/PromptShortcutModal.tsx`), which lists cron-scheduled prompt runs via `node-cron` (`src/lib/scheduler.ts`). Schedules persist to `.pfv-schedules.json` (gitignored) and are re-registered on every server start. Timezone: America/Sao_Paulo.

CRUD via `/api/schedules`: `GET` (list), `POST` (create), `PUT` (update), `DELETE` (remove), `PATCH` (run now). Each tick calls `runShortcut(prompt, { autoClose: true })` in `src/lib/runShortcut.ts`, which writes a temp `.ps1` that `Set-Location`s to the configured `resourcesPath` and spawns `cmd /c start "" powershell -File <script>` to run `claude "<prompt>"`. Cron only fires while the Next.js server is running.

No session tracking or status reporting ‚Äî agent lifecycle is handled elsewhere. Windows-only (returns error on other platforms).

### Budget bucket definitions

The three buckets map categories as follows:
- **Custos Fixos (30%)**: Housing, Health, Insurance, Groceries, Transportation
- **Conforto (25%)**: Wellness, Subscriptions, Personal Care, Services, Food/Dining, Recreation, Shopping, Travel, Family Support, Education, and any unlisted category (catch-all)
- **Liberdade Financeira (45%)**: `Investment (Troco Turbo)` + Net

`Investment (Troco Turbo)` must be the exact key in `expenses.by_category` for it to appear correctly in the liberdade_financeira bucket.
